== Browse graph-test-data and auto-convert via graphinout — DONE

The compare app can browse the
link:https://github.com/Calpano/graph-test-data[graph-test-data] repo and run engines over
files in *any* registry format, converting them to grale on the fly.

* *Path → format.* `scripts/runner.mjs` `formatFamily()` reads the family from the
  graph-test-data path convention (`/<category>/<family>/…`); a `grale` family is read
  natively, anything else is converted.
* *Convert.* via the graphinout `gio` CLI: `gio convert <file> --to grale` (overridable with
  the `GRAPHINOUT` env, or the `GIO` path). graphinout now ships a grale reader+writer.
* *Normalise.* `normalizeGrale()` fills what a faithful conversion omits so the result is
  layout-ready: a graph `value` label, node `width`/`height` defaults (sized from the
  label), and `meta.label` for the renderer. (Intentional — kept on the consumer side, not
  a graphinout-writer change.)
* *Browse.* `GET /api/browse` + the app's file-tree picker navigate graph-test-data and set
  a folder as the run input. `--INVALID…` test files are skipped by the runner.

Residual / nice-to-have:

* Per-file selection in the browser (currently you pick a *folder*).
* Optional `--from <reader>` from the path family for ambiguous extensions (graphinout's
  auto-detect covers the common cases today).
* graphinout puts source attributes (colour, etc.) outside `meta`; they surface as
  `UNKNOWN_FIELD` warnings. Harmless, but a tidier mapping into `meta` would be nicer.
